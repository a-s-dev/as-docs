[−][src]Function webext_storage::api::wipe_all

pub fn wipe_all(tx: &Transaction) -> Result<()>

While this API isn't available to extensions, Firefox wants a way to wipe all data for all addons but not sync the deletions. We also don't report the changes caused by the deletion. That means that after doing this, the next sync is likely to drag some data back in - which is fine. This is much like what the sync support for other components calls a "wipe", so we name it similarly.
